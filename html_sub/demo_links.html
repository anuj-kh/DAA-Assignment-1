Discussion on the algorithm:

  • The two problems - The Measure problem and the Contour problem-required the use of the data structure stripes, which were calculated using the STRIPES algorithm.
  • The STRIPES algorithm takes the edge array and the x and y intervals as input.
  • When the number of edges is 1, it computes:
    • A set of intervals L and R containing the y-projections from either the left or the right edge.
    • A set of coordinates P containing the y-projections of all the endpoints.
    • The stripes S with x interval taken from the input and y interval calculated using set of coordinates P
  • When the number of edges greater than is 1:
    • It divides the edges and the interval into 2 parts and recursively calls STRIPES 2 times as a part of the conquer step.
    • It computes sets L, R, P in the merge step of the algorithm using the L, R, and P from both the parts.
    • Then a copy function is called on both the pieces so that the different y-intervals in both the steps are made equal.
    • The blacken function updates the x_union field of both stripes to indicate that a rectangle entirely covers them.
    • The concat function merges both the stripes into a single one.
  • After the STRIPES function is completed, the measure is calculated using the x_union and y_interval fields of the stripe.
  • Contour_peices is the skeleton algorithm that is taking care of calculation of horizontal contour:
    • It first calculates the intervals of stripes that are present in the tree after that it calculates the required horizontal contour using the intersection , union and difference techniques as specified by the paper
    • After the calculation of horizontal contour, we are calculating the vertical contour lines. We are first merging the overlapping contour edges if they are of the same type(both bottom/both top). Otherwise we are leaving them as it is and after merging the contour lines we are taking out the end points of the final horizontal contour line and storing them in an array and sorting them and after that we are combining the 2 continuous points to form vertical contour lines
    • After the calculation of contour lines we are calculating the length of contour

Complexity(Asymptotic) analysis:

  • Let T(n) denote the worst case time for the algorithm.
  • Basic traversals and all take time O(n).
  • The copy, blacken and concat functions take time O(n2)
  • And we call STRIPES 2 times recursively.
  • Therefore, T(n)= 2 T(n/2) + O(n2)
  • On solving the following recursive relation, we got our time complexity as T(n)= O(n2 log n)

References:

    Güting, R.H. Optimal divide-and-conquer to compute measure and contour for a set of iso-rectangles. Acta Informatica 21, 271–291 (1984). https://doi.org/10.1007/BF00264251
